# Basic PySpark Pipeline

This is a small phase-wise PySpark pipeline for learning the basics.

It uses one static dummy input file:

```text
data/raw/customers.csv
```

No dummy data is generated by code.

## Flow

```text
Phase 1: Create SparkSession
Phase 2: Ingest customers.csv
Phase 3: Clean and transform customer fields
Phase 4: Validate data quality and split good/bad rows
Phase 5: Write good rows and bad rows to separate folders
```

## Folder Structure

```text
basic-pyspark-pipeline/
├── data/
│   ├── raw/customers.csv
│   ├── good/
│   └── bad/
├── src/
│   ├── phase_01_spark/spark_session.py
│   ├── phase_02_ingestion/ingest_customers.py
│   ├── phase_03_transformation/transform_customers.py
│   ├── phase_04_validation/validate_customers.py
│   ├── phase_05_loading/write_outputs.py
│   └── main.py
├── tests/
├── requirements.txt
└── README.md
```

## What Each File Does

- `src/phase_01_spark/spark_session.py`: Creates the local SparkSession.
- `src/phase_02_ingestion/ingest_customers.py`: Reads the raw CSV with an explicit schema.
- `src/phase_03_transformation/transform_customers.py`: Cleans text, casts data types, parses dates, and creates `customer_segment`.
- `src/phase_04_validation/validate_customers.py`: Finds missing IDs, duplicates, invalid emails, invalid ages, bad dates, and bad amounts.
- `src/phase_05_loading/write_outputs.py`: Writes good and bad rows to separate CSV output folders.
- `src/main.py`: Runs all phases in order.
- `tests/test_validate_customers.py`: Tests the validation split.

## Run

```bash
pip install -r requirements.txt
python src/main.py
```

Run tests:

```bash
pytest -q
```

## Output

Good rows are written here:

```text
data/good/customers_good/
```

Bad rows are written here:

```text
data/bad/customers_bad/
```

The bad output includes a `validation_error` column explaining why each row failed.
