# Phase 6 - AI Integration With OpenAI

## New Topic

OpenAI API integration, structured AI output, AI-generated resume
recommendations, interview preparation, answer strategy, study plan, and
recruiter pitch.

This phase extends Phase 5. The keyword matching, semantic matching, database
history, pgvector-ready embedding storage, frontend, and tests still work. The
new feature is an AI career coach layer.

## What The AI Generates

For every resume analysis, the backend asks OpenAI to generate:

- AI fit summary
- Resume improvement recommendations
- Job alignment advice
- Likely interview questions
- Why each interview question matters
- Strong answer bullet points
- Interview answer strategy
- 7-day study plan
- Recruiter pitch

Phase 6 requires `OPENAI_API_KEY` for analysis. If the key is missing, the API
returns a clear error instead of generating static placeholder questions or
suggestions.

## Why This Is Interview-Worthy

Students can explain that Phase 6 combines deterministic backend scoring with
LLM-generated coaching. The scoring engine calculates transparent numbers, while
OpenAI generates personalized language-heavy guidance that is harder to build
with rules alone.

## OpenAI Implementation

The AI integration is in:

```text
backend/app/services/ai_coach.py
```

It uses the OpenAI Responses API with Pydantic structured output. The schema is:

```text
backend/app/schemas/ai.py
```

This makes the model return predictable JSON fields instead of free-form text.
That is useful because the frontend and database can safely store and display
the generated interview kit.

The deterministic backend only provides factual inputs to the model:

- cleaned resume text
- cleaned job description
- matched skills
- missing skills
- score breakdown

The human-facing coaching fields are generated by OpenAI:

- `ai_summary`
- `resume_recommendations`
- `job_alignment_advice`
- `interview_questions`
- `answer_strategy`
- `study_plan`
- `recruiter_pitch`

## Interview Explanation

"In Phase 6, I added an AI coaching layer using the OpenAI API. The deterministic
part of the system still extracts skills, calculates semantic similarity, and
computes a weighted score. Then I send the cleaned resume, job description,
matched skills, missing skills, and score breakdown to an AI coach service. The
AI returns structured output containing resume recommendations, likely interview
questions, answer strategies, a study plan, and a recruiter pitch. I used a
separate service class so the OpenAI integration is isolated, testable, and easy
to replace."

## Common Interview Questions

### Why add OpenAI if you already have scoring?

Scoring is best for transparent numeric evaluation. OpenAI is better for
language-heavy coaching, like rewriting advice, interview preparation, and
personalized study planning.

### How do you avoid random unstructured AI output?

The service uses a Pydantic schema for structured output, so the response must
match fields such as `interview_questions`, `answer_strategy`, and
`recruiter_pitch`.

### What data do you send to OpenAI?

The cleaned resume text, cleaned job description, matched skills, missing
skills, score breakdown, and baseline suggestions.

### What if the API key is missing?

The API returns `503` with a clear message. Phase 6 intentionally does not
generate local static interview content because the goal of this phase is real
AI-generated coaching.

### How would you improve this phase?

Add streaming output, store prompt versions, add user accounts, add cost
tracking, add safety checks for sensitive resume data, and compare model output
quality with human feedback.
